# Code of Conduct

## Our commitment

We are committed to fostering a respectful, thoughtful, and intellectually honest
environment for everyone participating in the AIC-EdgeOS project.

This project values:
- Technical rigor
- Ethical awareness
- Mutual respect
- Constructive disagreement

---

## Expected behavior

Participants are expected to:

- Communicate with clarity and respect
- Critique ideas, not individuals
- Be open to being wrong
- Acknowledge uncertainty
- Respect different levels of experience

Strong opinions are welcome. Personal attacks are not.

---

## Unacceptable behavior

The following behaviors are not tolerated:

- Harassment, discrimination, or hostility
- Dismissive or condescending language
- Personal attacks disguised as technical critique
- Persistent bad-faith argumentation
- Pressure to compromise ethical or safety principles

---

## Technical disagreements

Disagreement is natural and encouraged.

When disagreeing:
- Provide reasoning, not authority
- Prefer evidence over rhetoric
- Accept that some decisions are value-based
- Respect final decisions even if you disagree

---

## Scope

This Code of Conduct applies to:
- GitHub issues and pull requests
- Discussions and documentation
- Any official project communication channels

---

## Enforcement

Project maintainers are responsible for enforcing this Code of Conduct.

Actions may include:
- Requesting clarification or tone adjustment
- Closing discussions
- Rejecting contributions
- Temporary or permanent bans (in severe cases)

All enforcement actions will be proportionate and documented internally.

---

## Final note

This project exists to solve hard problems.
Hard problems require calm minds, steady hands, and mutual respect.
